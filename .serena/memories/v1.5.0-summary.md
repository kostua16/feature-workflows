# Milestone v1.5.0 — Project-Scale Extract Design (SHIPPED 2026-07-22)

Turned the v1.4.5 extract flow into **trustworthy whole-project extraction**, and extended the same
durability/truthfulness/bounded-execution contracts to `/design-feature` and the shared engine.
Command surface is UNCHANGED (still 8 commands, 31 agents). See `mem:handoff` for current position.

## Scope & quality
- 11 phases (1–7 extract-orchestration core; 8–11 design-mode extension) + a post-audit tech-debt cleanup.
- 36/36 requirements delivered, Nyquist-validated, UAT-verified GOAL MET.
- 1470 tests, build drift-free. **9 real defects caught** across validation (8) + UAT (1) — all fixed.
- Milestone audit status: **passed**.

## Headline capabilities
- **Whole-project extraction from one `/extract-design`**: bounded, durable, resumable per-feature
  segments via a top-level orchestrator + `fp-extract-slice` leaf (one-level composition). Budgeted
  admission w/ non-spendable reserve, retry policy, failure isolation, transactional (monotonic/idempotent)
  continuation. Never silently loses work or overstates completion.
- **Truthful status**: `extractReady`/`designReady` set only when genuinely earned; handoff + read-only
  status share one immutable projection.
- **Design-mode hardening**: gate-level durable checkpoints + auto-recovering state writes (last-good
  snapshot), enforced per-gate/per-loop budgets + bounded prompts, transient-error backoff, deterministic
  digest-based artifact verification.
- **v1.4.5 compat**: older state hydrates safely; `--resume --migrate` upgrades a legacy extract state file.

## Key primitives added (src modules)
lifecycle, migration, revision, inventory, discovery, graph-validation, queue-semantics, schedulability,

## Known/deferred

  libraries only** — the runtime extract path uses an LLM `subsystem-decomposer` + `seedExtractQueue` by
  design (scoped in `02-PLAN.md`). Runtime determinism was NOT a hard requirement.
- Token-budget characterization numbers not measured (plumbing added; needs a dogfood run).
- Milestone tagged `v1.5.0` LOCALLY (not pushed). Marketplace/plugin.json still pinned at 1.4.5 — release/pin is a separate step.
